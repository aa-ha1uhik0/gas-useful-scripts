
# はじめに
社内のGoogleフォームやスプレッドシートの更新をトリガーにして、Slackへ通知を送る仕組みを作っている方は多いのではないでしょうか。
その際、「申請者のメールアドレスはあるけれど、Slackでその人に **自動で@メンション** を飛ばしたい」「Slackの **プロフィール画像（アバター）** を通知に埋め込みたい」という場面によく遭遇します。

そこで、メールアドレスをキーにしてSlackのユーザー情報を一撃で逆引きし、APIのレスポンスを丸ごと呼び出し元に返す、軽量で汎用的なGAS（Google Apps Script）のコードスニペットを作成しました！
私はGASライブラリとして自身向けにデプロイして様々なGASプロジェクトから利用しています。

ご自身のGASプロジェクトにコピペするだけですぐに使え、トークンなどの認証情報を関数内に持たせないクリーンな設計にしています。

---

# 1. 設計のこだわり（レスポンスを「丸ごと」返す）

このスクリプトは、Slack公式の `users.lookupByEmail` APIをシンプルにラップしたものです。

よくあるサンプルコードでは、関数内で処理して「ユーザーID（`Uxxxxxx`）」だけを返すものが多いですが、本スクリプトは**Slack APIから返ってきたJSONオブジェクトをパースして丸ごと返却**します。

これにより、呼び出し側の目的（ユースケース）に応じて、以下のように自由にデータを引っこ抜くことができます。
* Aシステムでは： **ユーザーID** を取得して自動メンションに利用
* Bシステムでは： **表示名（`display_name`）** を取得して台帳の転記に利用
* Cシステムでは： **アバター画像URL（`image_512`）** を取得してBlock Kitのアイコンに利用

検索関数側を一切修正することなく、呼び出し側のGASだけで100%コントロールできる高い拡張性を備えています。

# 2. 事前準備（Slackアプリの設定）

Slack API側でメールアドレス検索の権限（Scope）を持ったBotトークンが必要です。

1. **Slack Appの作成**
   [Slack API (My Apps)](https://api.slack.com/apps) にアクセスし、`Create New App` からアプリを作成します。
2. **権限（Scopes）の付与**
   左メニューの **`OAuth & Permissions`** を開き、下部にある **`Scopes`** > **`Bot Token Scopes`** に以下の2つの権限を追加します。
   * `users:read` （ユーザー情報の読み取り）
   * `users:read.email` （メールアドレスによる検索・読み取りに必須）
3. **アプリのインストールとトークンの取得**
   画面上部に戻り、`Install to Workspace` をクリックしてワークスペースにアプリをインストール（許可）します。
   インストール完了後、画面に表示される `xoxb-` から始まる **`Bot User OAuth Token`** をコピーして控えておきます。

:::note warn
**注意：** `users:read.email` スコープを後から追加した場合は、必ずアプリを再インストール（Reinstall App）してください。再インストールしないとAPI呼び出し時に `missing_scope` エラーが発生します。
:::

# 4. 実装コード例（使い方）

セキュリティの観点から、SlackのBotトークンはコード内に直接書かず、GASの **「スクリプトプロパティ」** に保存して利用することを強く推奨します。

### 1. スクリプトプロパティの設定
GASエディタの左メニュー「プロジェクトの設定（歯車マーク）」＞「スクリプトプロパティ」に以下を追加します。
* プロパティ名: `SLACK_BOT_TOKEN`
* 値: （先ほど控えた `xoxb-` から始まるトークン）

### 2. 呼び出し側のサンプルスクリプト
以下は、メインの処理（フォーム送信時など）から先ほどの検索関数を呼び出し、メンション文字列やアバターURLを抽出する実装例です。

```javascript
/**
 * メールアドレスからSlackユーザーを検索するテスト関数
 */
function testSlackUserLookup() {
  // 1. スクリプトプロパティから安全にBotトークンを取得
  const botToken = PropertiesService.getScriptProperties().getProperty('SLACK_BOT_TOKEN');
  
  // 2. 検索したいメールアドレスを指定（例: Googleフォームから自動取得したメアドなど）
  const targetEmail = 'example-user@yourdomain.com';

  if (!botToken) {
    console.error('エラー: スクリプトプロパティに SLACK_BOT_TOKEN が設定されていません。');
    return;
  }

  // 3. コピペした関数を呼び出す（トークンとメアドを引数として渡す）
  const response = getUserByEmail(botToken, targetEmail);

  // 4. 呼び出し元でレスポンスのステータスと中身をチェック
  if (response && response.ok) {
    // --- 呼び出し元の目的に応じて、好きな項目を自由に抽出可能！ ---
    
    const slackUserId = response.user.id;                        // メンバーID (例: U07B1Q6SS6R)
    const realName = response.user.profile.real_name;            // 本名
    const displayName = response.user.profile.display_name;      // 表示名（ニックネーム）
    const avatarUrl = response.user.profile.image_512;           // プロフィール画像(512x512)のURL

    console.log(`🎉 ユーザー特定成功！: ${realName} (@${displayName})`);
    console.log(`🆔 SlackメンバーID: ${slackUserId}`);
    console.log(`💬 メンション用文字列: <@${slackUserId}>`);
    console.log(`🖼️ アバターURL: ${avatarUrl}`);

  } else {
    // APIの成否、またはエラー内容を呼び出し元でハンドリング
    const errorCode = response ? response.error : '通信エラー';
    console.warn(`❌ ユーザーが見つからないか、エラーが発生しました。理由: ${errorCode}`);
  }
}
```
# 5. 主なレスポンス構造（参考）
`response.ok` が `true` の場合、以下のようなオブジェクト構造が丸ごと返ってきます（一部抜粋）。詳細なスキーマは [Slack公式ドキュメント](https://api.slack.com/methods/users.lookupByEmail) をご参照ください。

```json
{
  "ok": true,
  "user": {
    "id": "W012A3C45",
    "name": "beautifuljones",
    "profile": {
      "real_name": "Egon Spengler",
      "display_name": "Spengler",
      "email": "spengler@ghostbusters.example.com",
      "image_512": "https://..."
    }
  }
}
```
# 6. よくあるトラブルシューティング

### ① `users_not_found` エラーになる
* **原因:** 指定したメールアドレスを持つユーザーがSlackワークスペース内に存在しない、またはアドレスが完全に一致していません。
* **対策:** Slackに登録されているメールアドレス（大文字小文字、エイリアス等）と一致しているか確認してください。

### ② `missing_scope` エラーになる
* **原因:** Slackアプリに `users:read.email` の権限が付与されていないか、付与した後にアプリを「再インストール」していません。
* **対策:** Slack APIの管理画面でScopeを確認し、画面上部の `Reinstall App` を実行してください。

### ③ `invalid_auth` エラーになる
* **原因:** 渡している `SLACK_BOT_TOKEN` の値が間違っているか、正しくスクリプトプロパティから取得できていません。
* **対策:** プロパティ名に余計なスペースが含まれていないか、値が `xoxb-` から始まっているか確認してください。

:::note warn
**補足：** 大量データの一括処理について
本API（`users.lookupByEmail`）は、Slackの`Rate Limits`（Tier 3: 1分間に約50回まで）の対象です。スプレッドシートのループ処理などで短時間に大量の検索を行うと `429 Too Many Requests` エラーになるため、連続実行する場合は `Utilities.sleep()` 等で適切な待機時間を設けてください。
:::

# 7. 利用上の注意・セキュリティ（重要）

外部のAPIを連携するスクリプトをプロジェクトに導入する際は、セキュリティリスクを正しく理解しておく必要があります。本コードを利用するにあたり、以下の点にご注意ください。

### 🔑 1. トークンの適切な管理と最小権限の原則
関数には、引数としてSlackのBotトークン（`xoxb-...`）を渡す設計になっています。コード内にトークンを直接記述せず、必ずGASのスクリプトプロパティ等を利用して隠蔽してください。
また、発行するSlack Botには、本処理が必要とする `users:read` および `users:read.email` 以外の余剰なスコープ（メッセージ送信用の `chat:write` など）を同一のトークンに混在させないようにし、万が一の漏洩時の被害を最小限に抑えるようにしてください。

### 👥 2. 個人情報（PII）の取り扱いについて
利便性を高めるため、Slack APIからのレスポンスを加工せず丸ごと返却する仕様にしています。
レスポンスオブジェクトには、ユーザーのメールアドレス、氏名、プロフィールなどの個人情報（PII）が多数含まれています。`console.log(response)` のようにオブジェクト全体をそのまま実行ログへ出力したまま放置すると、ログ経由で個人情報が漏洩するリスクが生じます。デバッグ後はログ出力を削除するか、必要なプロパティのみを絞り込んで扱うように実装してください。
